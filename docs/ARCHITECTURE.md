# MyMoney 아키텍처 문서

## 1. 프로젝트 개요

**MyMoney**는 AI 기반 가계부 서비스이다.

- **핵심 기능**
  - OCR 자동 전표 생성: 영수증 촬영 → 자동 분류 → 전표 생성
  - 카드사/국세청 연동 자동 수집: 거래 내역 자동 수집 → 분류 → 전표 생성

---

## 2. 기술 스택 결정 사항 및 근거

| 영역 | 기술 | 근거 |
|------|------|------|
| **프론트엔드** | Flutter (Dart) | 1인/소규모 팀에서 iOS, Android, Web, Desktop 크로스플랫폼 대응에 최적 |
| **백엔드 (메인)** | C# (ASP.NET Core) | OOP + 성능 균형. Windows 로컬 서버 친화적이며 엔터프라이즈급 안정성 |
| **백엔드 (AI)** | Python (FastAPI) | AI/ML 생태계 독점적 지위. OCR 후처리, 자동 분류 모델 서빙 |
| **DB** | MySQL | 언어 중립적, 로컬 서버 운용에 적합, 충분한 생태계 |

**레퍼런스 아키텍처**: 뱅크샐러드 (Go + Kotlin + Python) 경량화 버전

---

## 3. 아키텍처 설계

### 3.1 전체 시스템 구조

```
┌─────────────┐     REST API     ┌──────────────────┐     Internal API     ┌─────────────────┐
│   Flutter    │ ◄─────────────► │   C# API Server  │ ◄─────────────────► │  Python AI 서버  │
│  (클라이언트) │                 │  (ASP.NET Core)  │                     │   (FastAPI)      │
└─────────────┘                  └────────┬─────────┘                     └─────────────────┘
                                          │
                                          │ SQL
                                          ▼
                                   ┌─────────────┐
                                   │    MySQL     │
                                   │    (DB)      │
                                   └─────────────┘
```

### 3.2 입력 경로 A: OCR 자동 전표 생성

```
영수증 촬영
    │
    ▼
온디바이스 ML Kit (OCR)
    │
    ▼
규칙 엔진 (텍스트 파싱)
    │
    ▼
전표 생성
```

- 온디바이스에서 ML Kit으로 1차 OCR 수행
- 추출된 텍스트를 규칙 엔진으로 파싱하여 금액, 일시, 가맹점 추출
- 분류 후 전표 자동 생성

### 3.3 입력 경로 B: 카드사/국세청 연동

```
카드사 / 국세청 API
    │
    ▼
서버 수집 엔진
    │
    ▼
규칙 엔진 / AI 분류
    │
    ▼
전표 생성
```

- 서버에서 카드사/국세청 API를 통해 거래 내역 자동 수집
- 규칙 엔진과 AI 모델을 병행하여 카테고리 분류
- 분류 결과를 기반으로 전표 생성

### 3.4 공통 파이프라인

```
입력 경로 A (OCR)  ──┐
                      ├──► 분류 엔진 ──► 전표 생성 ──► DB 저장
입력 경로 B (연동)  ──┘
```

- 두 경로 모두 동일한 **분류 → 전표 생성** 흐름으로 합류
- 분류 엔진은 규칙 기반 + AI 모델 하이브리드

## 4. 단계별 빌드업 로드맵

### 1단계: MVP

```
Flutter ◄──► C# (단일 서버 + 내장 규칙 엔진) ◄──► MySQL
```

- Flutter 클라이언트 + C# 단일 서버 구성
- 규칙 기반 분류를 C# 서버 내장으로 처리
- 수동 입력 + 기본 OCR 지원

### 2단계: AI 고도화

```
Flutter ◄──► C# API Server ◄──► Python AI Server
                  ▲
                  │
               MySQL
```

- Python AI 서버 추가
- 자동 분류 고도화 (ML 모델 서빙)
- 카드사/국세청 연동 기능 추가

### 3단계: 스케일 아웃

```
Flutter ◄──► C# API Server ◄──► Python AI Server
                  ▲                    ▲
                  │                    │
               MySQL            Go 수집 엔진
```

- Go 수집 엔진 분리 (고성능 동시성 처리)
- 수십만 동접 대응 가능한 아키텍처
- 마이크로서비스 전환 완료

## 5. 저장소 전략

- **현재**: 모노레포 (프로젝트 루트에서 `app/`, `server/` 폴더 분리)
  - 1인 개발 단계에서 관리 편의성 극대화
  - 공유 타입/스키마 동기화 용이
- **향후**: `git subtree split`으로 저장소 분리
  - 협업 규모 확대 시 독립 배포 파이프라인 구축
  - 각 서비스별 독립적인 CI/CD 운영
